## 📖 Описание лабораторной работы

**FilmHub** — это веб-приложение средней сложности, представляющее собой каталог фильмов с возможностью оставлять отзывы и оценки. Зарегистрированные пользователи могут писать рецензии на фильмы, а администраторы — управлять каталогом.

### Цель работы

Закрепить навыки разработки серверной части веб-приложения на PHP с использованием MySQL, отработать механизмы аутентификации, работы с формами, валидацией, сессиями и ролевой моделью.

### Основные этапы

1. Проектирование структуры БД и реляционной модели.
2. Реализация системы аутентификации с хешированием паролей.
3. Реализация публичного компонента с динамическим контентом.
4. Реализация форм создания ресурса и поиска с двухсторонней валидацией.
5. Реализация защищённого компонента с ролью администратора.
6. Применение мер безопасности (CSRF, XSS, SQL-инъекции).
7. Тестирование и оформление документации.

## 🗄 Структура базы данных

База данных называется `filmhub`. Состоит из 4 таблиц:

### Таблица `users` — пользователи системы

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT, AUTO_INCREMENT | Первичный ключ |
| username | VARCHAR(50), UNIQUE | Логин |
| email | VARCHAR(100), UNIQUE | Email |
| password_hash | VARCHAR(255) | Хеш пароля (bcrypt) |
| role | ENUM('user','admin') | Роль |
| created_at | TIMESTAMP | Дата регистрации |

### Таблица `genres` — жанры

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT, AUTO_INCREMENT | Первичный ключ |
| name | VARCHAR(50), UNIQUE | Название жанра |

### Таблица `films` — фильмы

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT, AUTO_INCREMENT | Первичный ключ |
| title | VARCHAR(150) | Название фильма |
| director | VARCHAR(100) | Режиссёр |
| release_year | INT | Год выпуска |
| genre_id | INT, FK→genres | Жанр |
| duration | INT | Длительность в минутах |
| description | TEXT | Описание |
| poster | VARCHAR(255) | URL постера |
| is_featured | TINYINT(1) | Показывать на главной |
| created_at | TIMESTAMP | Дата добавления |

### Таблица `reviews` — отзывы

| Поле | Тип | Описание |
|------|-----|----------|
| id | INT, AUTO_INCREMENT | Первичный ключ |
| user_id | INT, FK→users | Автор отзыва |
| film_id | INT, FK→films | Фильм |
| rating | INT (1-10) | Оценка |
| comment | TEXT | Текст отзыва |
| would_recommend | TINYINT(1) | Рекомендация |
| watched_date | DATE | Дата просмотра |
| created_at | TIMESTAMP | Дата создания |

**Связи:**

- `films.genre_id` → `genres.id` (один-ко-многим)
- `reviews.user_id` → `users.id` (один-ко-многим, ON DELETE CASCADE)
- `reviews.film_id` → `films.id` (один-ко-многим, ON DELETE CASCADE)
- Уникальная пара `(user_id, film_id)` в `reviews` — один пользователь может оставить только один отзыв на фильм.

## 🚀 Инструкция по запуску

### Вариант А — через MAMP (macOS) ⭐ рекомендуемый для Mac

1. **Установите MAMP** с официального сайта: https://www.mamp.info/en/downloads/
2. Скопируйте/клонируйте папку проекта в `/Applications/MAMP/htdocs/filmhub`:

   ```bash
   cd /Applications/MAMP/htdocs
   git clone <url-репозитория> filmhub
   ```

3. Откройте приложение **MAMP** и нажмите **Start Servers** (Apache + MySQL должны запуститься).
4. Откройте **phpMyAdmin** по адресу http://localhost:8888/phpMyAdmin/
5. Создайте новую БД с именем `filmhub` (кодировка `utf8mb4_unicode_ci`) → выберите её → вкладка **Import** → выберите файл `sql/filmhub.sql` → **Go**.
6. Откройте файл `config/config.php` и убедитесь, что значения соответствуют MAMP:

   ```php
   define('DB_HOST', 'localhost');     // если MySQL на стандартном порту
   define('DB_USER', 'root');
   define('DB_PASS', 'root');           // пароль MySQL у MAMP по умолчанию — 'root'
   define('SITE_URL', 'http://localhost:8888/filmhub');
   ```

   > Если получаете ошибку "Connection refused" — поменяйте `DB_HOST` на `'localhost:8889'` (порт MySQL у MAMP).

7. Откройте в браузере http://localhost:8888/filmhub/

### Вариант Б — через XAMPP (Windows / Linux)

1. **Установите XAMPP** с официального сайта: https://www.apachefriends.org/
2. Запустите **Apache** и **MySQL** через панель управления XAMPP.
3. Скопируйте/клонируйте проект в `C:/xampp/htdocs/filmhub`.
4. Откройте **phpMyAdmin** по адресу http://localhost/phpmyadmin → создайте БД `filmhub` → импортируйте `sql/filmhub.sql`.
5. В `config/config.php` оставьте настройки по умолчанию (для XAMPP `root` без пароля):

   ```php
   define('DB_PASS', '');
   define('SITE_URL', 'http://localhost/filmhub');
   ```

6. Откройте http://localhost/filmhub/

### Вариант В — через встроенный сервер PHP

> Подходит, если PHP установлен глобально, а MySQL работает отдельно.

1. Создайте базу и импортируйте схему:

   ```bash
   mysql -u root -p < sql/filmhub.sql
   ```

2. Перейдите в папку проекта и запустите встроенный сервер:

   ```bash
   cd filmhub
   php -S localhost:8000
   ```

3. В `config/config.php` укажите:

   ```php
   define('SITE_URL', 'http://localhost:8000');
   ```

4. Откройте http://localhost:8000/

### Все настройки в одном месте

В файле `config/config.php`:

```php
define('DB_HOST', 'localhost');     // или 'localhost:8889' для MAMP
define('DB_NAME', 'filmhub');
define('DB_USER', 'root');
define('DB_PASS', 'root');           // 'root' для MAMP, '' для XAMPP
define('DB_CHARSET', 'utf8mb4');
define('SITE_URL', 'http://localhost:8888/filmhub'); // под ваш сервер
```

## 🔒 Безопасность

В проекте реализованы следующие меры безопасности:

| Угроза | Защита |
|--------|--------|
| **SQL-инъекции** | Все запросы к БД выполняются через PDO с подготовленными выражениями (prepared statements) |
| **XSS (Cross-Site Scripting)** | Все выводимые в HTML данные проходят через функцию `e()` (htmlspecialchars) |
| **CSRF (Cross-Site Request Forgery)** | Все формы и опасные GET-действия защищены случайным токеном (`csrf_token`), хранящимся в сессии |
| **Хранение паролей** | Пароли хешируются через `password_hash()` (bcrypt) и проверяются `password_verify()` |
| **Session fixation** | После успешного входа выполняется `session_regenerate_id(true)` |
| **Привилегии** | Доступ к админ-зоне защищён функцией `requireAdmin()`, проверяющей сессию и роль |
| **Валидация ввода** | Двойная валидация — на клиенте (JS) и обязательно на сервере |
| **Целостность данных** | Внешние ключи на уровне БД, проверка диапазонов в коде |

## ❓ Ответы на контрольные вопросы

**1. Что такое сессии в PHP и зачем они нужны?**
Сессии — это механизм, позволяющий сохранять данные пользователя между разными HTTP-запросами. PHP создаёт уникальный идентификатор сессии (хранится в cookie клиента) и связывает с ним данные на сервере. В FilmHub сессии используются для хранения id, логина и роли авторизованного пользователя, а также CSRF-токена и flash-сообщений.

**2. Чем отличаются prepared statements от обычных SQL-запросов и почему они безопаснее?**
В обычном запросе данные пользователя вставляются прямо в строку SQL — это уязвимо для SQL-инъекций. В prepared statements запрос отправляется в БД с плейсхолдерами (`?` или `:name`), а параметры передаются отдельно. БД сама подставляет их с экранированием, поэтому злоумышленник не может «сломать» запрос.

**3. Почему пароли нельзя хранить в открытом виде или с обычным MD5/SHA1?**
Если БД утечёт, открытые пароли станут известны напрямую, а MD5/SHA1 — взламываются перебором или радужными таблицами за секунды. `password_hash()` использует bcrypt с автоматическим солением и регулируемой стоимостью вычисления (cost), что делает перебор паролей вычислительно дорогим.

**4. Что такое XSS и как от него защищаться?**
XSS (Cross-Site Scripting) — это внедрение злоумышленного JS в контент сайта (например, в комментарий). Когда другой пользователь открывает страницу, скрипт выполняется в его браузере. Защита: экранировать все выводимые в HTML данные через `htmlspecialchars()` (в проекте — функция `e()`).

**5. Что такое CSRF и как его предотвратить?**
CSRF — атака, при которой злоумышленник заставляет браузер пользователя отправить запрос на сайт, где тот авторизован (например, удалить аккаунт через скрытую форму). Защита — добавление в каждую форму уникального токена, привязанного к сессии. Сервер проверяет, что присланный токен совпадает с сохранённым.

**6. В чём смысл архитектуры MVC?**
MVC разделяет код на три слоя: Model (работа с данными), View (отображение), Controller (логика обработки запросов). Это упрощает поддержку, повторное использование и тестирование кода. В FilmHub применена «лёгкая» модульная архитектура: общие функции вынесены в `includes/`, форматирование — в шапку/футер, страницы выступают одновременно контроллером и видом (упрощённый подход для учебного проекта).

**7. Зачем нужна двойная валидация (клиент + сервер)?**
Клиентская валидация удобна для пользователя — она моментальна и не нагружает сервер. Но её легко обойти (отключить JS, отправить запрос напрямую). Поэтому серверная валидация обязательна как защита от любых вредоносных или некорректных данных.

**8. Что делает функция `password_hash()` под капотом?**
По умолчанию использует алгоритм bcrypt: генерирует случайную «соль», объединяет её с паролем, выполняет несколько раундов хеширования (по умолчанию 10) и возвращает строку, содержащую алгоритм, cost, соль и хеш. Для проверки используется `password_verify()` — она извлекает соль из хранимого хеша и повторяет операцию с введённым паролем.
